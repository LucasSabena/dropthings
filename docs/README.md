# DropThings Docs

DropThings is a native macOS utility hub for small productivity fixes that are annoying to manage as separate apps.

Start here:

- `product-plan.md`: vision, goals, non-goals, and success metrics.
- `roadmap.md`: build order from MVP to a polished modular app.
- `implementation-checklist.md`: granular steps agents can execute.
- `architecture.md`: app structure, module system, and boundaries.
- `modules.md`: planned modules and implementation notes.
- `design-system.md`: shared tokens and UI rules.
- `design-tokens.json`: machine-readable token seed.
- `permissions-security.md`: macOS permissions, risks, and safety model.
- `quality-plan.md`: tests, manual verification, release checks.
- `research/open-source-references.md`: apps and projects to study.
- `research/lazyweb.md`: Lazyweb workflow notes for product UI design.
- `skills.md`: project-local skills installed for future agents.
